### 表字段说明

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| user | 用户基础信息 | id, username, password_hash, status |
| user_profile | 用户扩展信息 | user_id, gender, signature, location |
| user_stat | 用户统计数据 | user_id, post_count, comment_count, follower_count, following_count |
| topic | 话题 | id, name, quote_num |
| post | 帖子主表 | id, user_id, comment_count, visibility |
| post_content | 帖子内容 | post_id, user_id, content, type, sort |
| post_topic | 帖子-话题关联 | post_id, topic_id |
| comment | 评论主表 | id, group_id, user_id, parent_id, level |
| comment_content | 评论内容 | comment_id, content |
| comment_vote | 评论投票 | comment_id, user_id, vote_type |
| following | 关注关系 | user_id, follow_id |

## 总结

### 关键设计模式

1. **API + RPC 双层架构**: API 层负责 REST 接口，RPC 层负责业务逻辑和数据库操作
2. **数据库事务**: 涉及多表操作的接口都使用事务保证一致性
3. **Kafka 事件驱动**: 帖子创建/删除通过 Kafka 异步更新用户统计
4. **软删除**: 核心表都使用 `deleted_at` 实现软删除

### 事务使用场景

| 接口 | 涉及表 | 原因 |
|------|--------|------|
| Register | user, user_profile, user_stat | 必须同时创建三个表记录 |
| CreatePost | post, post_content, topic, post_topic | 帖子内容、话题关联必须一致 |
| DeletePost | topic, post_topic, post_content, post | 删除操作必须按顺序回滚 |
| PostCreatedConsumer | user_stat | 计数更新需要事务保护 |
| PostDeletedConsumer | user_stat | 计数更新需要事务保护 |

### Kafka 主题

| 主题 | 生产者 | 消费者 | 用途 |
|------|--------|--------|------|
| post-created | Content RPC | User RPC | 帖子创建，增加发帖数 |
| post-deleted | Content RPC | User RPC | 帖子删除，减少发帖数 |
