# SoChill 产品需求文档 - 前端开发指南

## 1. 项目概述

SoChill 是一个社交内容分享平台，采用微服务架构，包含用户、内容、评论、关系四个核心服务。

### 技术架构
- **前端**: Vue 3 + TypeScript + Element Plus + Vite
- **后端**: Go-Zero 微服务框架 + gRPC + Kafka + MySQL + Etcd
- **部署**: Docker Compose

---

## 2. 通用规范

### 2.1 响应格式规范
所有 API 响应统一遵循以下格式：

```typescript
interface BaseResponse {
  code: number;      // 0表示成功，其他表示错误
  msg: string;       // 响应消息
}
```

### 2.2 错误码规范
| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| 999 | 未知错误 |
| 1001 | 用户名或密码为空 |
| 1002 | 用户名已存在 |
| 1003 | 用户名或密码错误 |
| 1004 | 账号异常 |
| 1005 | 内部错误 |
| 1006 | 未授权访问 |
| 1007 | 参数错误 |
| 1008 | 用户不存在 |
| 1009 | 无效的刷新令牌 |
| 2001 | 不能关注自己 |
| 2002 | 已经关注了 |
| 2003 | 未关注 |
| 3001 | 帖子不存在 |
| 3002 | 没有操作权限 |
| 3003 | 帖子标题不能为空 |
| 3004 | 帖子内容不能为空 |
| 4001 | 评论不存在 |
| 4002 | 评论内容不能为空 |
| 4003 | 已经点赞了 |
| 4004 | 未点赞 |

### 2.3 认证规范
- **Access Token**: 在请求头中携带 `Authorization: Bearer ${accessToken}`
- **Token 过期**: Access Token 过期后，使用 Refresh Token 刷新
- **401 处理**: 收到 401 状态码时，自动跳转到登录页

## 7. 前端实现建议

### 7.1 状态管理 (Pinia Store)

```typescript
// stores/user.ts
import { defineStore } from 'pinia';
import { ref } from 'vue';

export const useUserStore = defineStore('user', () => {
  const token = ref<string>('');
  const refreshToken = ref<string>('');
  const userId = ref<number>(0);
  const userInfo = ref<UserInfo | null>(null);

  const setToken = (newToken: string) => {
    token.value = newToken;
    localStorage.setItem('token', newToken);
  };

  const setRefreshToken = (newToken: string) => {
    refreshToken.value = newToken;
    localStorage.setItem('refreshToken', newToken);
  };

  const setUserId = (id: number) => {
    userId.value = id;
  };

  const logout = () => {
    token.value = '';
    refreshToken.value = '';
    userId.value = 0;
    userInfo.value = null;
    localStorage.removeItem('token');
    localStorage.removeItem('refreshToken');
  };

  const init = () => {
    const savedToken = localStorage.getItem('token');
    const savedRefreshToken = localStorage.getItem('refreshToken');
    if (savedToken) token.value = savedToken;
    if (savedRefreshToken) refreshToken.value = savedRefreshToken;
  };

  return {
    token,
    refreshToken,
    userId,
    userInfo,
    setToken,
    setRefreshToken,
    setUserId,
    logout,
    init
  };
});
```

### 7.2 Token 刷新机制

在 request.ts 中实现自动刷新：

```typescript
let isRefreshing = false;
let refreshSubscribers: ((token: string) => void)[] = [];

function subscribeTokenRefresh(callback: (token: string) => void) {
  refreshSubscribers.push(callback);
}

function onRefreshed(token: string) {
  refreshSubscribers.forEach(callback => callback(token));
  refreshSubscribers = [];
}

service.interceptors.response.use(
  (response: AxiosResponse) => {
    const res = response.data;
    if (res.code !== 0) {
      if (res.code === 1006) {
        // 未授权，尝试刷新 token
        const originalRequest = response.config;
        if (!isRefreshing) {
          isRefreshing = true;
          const userStore = useUserStore();
          refreshToken({ refreshToken: userStore.refreshToken })
            .then((res) => {
              userStore.setToken(res.accessToken);
              userStore.setRefreshToken(res.refreshToken);
              onRefreshed(res.accessToken);
              isRefreshing = false;
            })
            .catch(() => {
              userStore.logout();
              window.location.href = '/login';
              isRefreshing = false;
            });
        }
        return new Promise((resolve) => {
          subscribeTokenRefresh((token) => {
            originalRequest.headers.Authorization = `Bearer ${token}`;
            resolve(service(originalRequest));
          });
        });
      }
      ElMessage.error(res.msg || '请求失败');
      return Promise.reject(new Error(res.msg || '请求失败'));
    }
    return res;
  },
  // ... 其他错误处理
);
```

### 7.3 分页加载实现

```typescript
// 帖子列表示例
import { ref } from 'vue';
import { getPostList } from '@/api/content';

export function usePostList() {
  const list = ref<PostInfo[]>([]);
  const loading = ref(false);
  const hasMore = ref(true);
  const page = ref(1);
  const pageSize = 20;

  const load = async (reset = false) => {
    if (loading.value || (!hasMore.value && !reset)) return;
    
    loading.value = true;
    if (reset) {
      page.value = 1;
      hasMore.value = true;
      list.value = [];
    }

    try {
      const res = await getPostList({
        page: page.value,
        pageSize
      });
      if (reset) {
        list.value = res.list;
      } else {
        list.value.push(...res.list);
      }
      hasMore.value = list.value.length < res.total;
      page.value++;
    } catch (error) {
      console.error('加载失败:', error);
    } finally {
      loading.value = false;
    }
  };

  return {
    list,
    loading,
    hasMore,
    load
  };
}
```

// 评论列表示例（游标分页）
import { ref } from 'vue';
import { getCommentList } from '@/api/comment';

export function useCommentList(postId: number) {
  const list = ref<CommentItem[]>([]);
  const loading = ref(false);
  const hasMore = ref(true);
  const cursor = ref(0);

  const load = async (reset = false) => {
    if (loading.value || (!hasMore.value && !reset)) return;

    loading.value = true;
    if (reset) {
      cursor.value = 0;
      hasMore.value = true;
      list.value = [];
    }

    try {
      const res = await getCommentList({
        groupId: postId,
        cursor: cursor.value,
        pageSize: 20,
        sortType: 'time'
      });
      if (reset) {
        list.value = res.list;
      } else {
        list.value.push(...res.list);
      }
      hasMore.value = res.list.length === 20;
      if (res.list.length > 0) {
        cursor.value = res.list[res.list.length - 1].root.id;
      }
    } catch (error) {
      console.error('加载评论失败:', error);
    } finally {
      loading.value = false;
    }
  };

  return {
    list,
    loading,
    hasMore,
    load
  };
}

## 10. 注意事项

1. **时间戳处理**: 后端返回的时间戳是毫秒，前端需要转换为 Date 对象
2. **空值处理**: 注意响应中可选字段可能为 null 或 undefined
3. **图片加载**: 使用懒加载和占位图提升用户体验
4. **防抖节流**: 搜索、输入等场景使用防抖节流
5. **错误重试**: 网络请求失败时可以适当重试
6. **加载状态**: 所有异步操作都需要显示加载状态
7. **分页边界**: 正确处理分页的边界情况
