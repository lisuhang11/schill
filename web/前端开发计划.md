# SChill 社交应用前端开发计划

## 一、项目概述

### 1.1 项目背景
SChill 是一个基于 go-zero 微服务架构构建的社交平台，包含用户管理、内容发布、评论互动和关注关系等核心功能。

### 1.2 技术栈
- **框架**: Vue 3 (Composition API) + TypeScript
- **UI 组件库**: Element Plus
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP 客户端**: Axios
- **构建工具**: Vite

### 1.3 项目目录结构
```
schill-frontend/
├── public/
│   └── favicon.ico
├── src/
│   ├── api/              # API 接口封装
│   ├── assets/           # 静态资源
│   ├── components/       # 公共组件
│   ├── router/           # 路由配置
│   ├── stores/           # Pinia 状态管理
│   ├── utils/            # 工具函数
│   ├── views/            # 页面组件
│   ├── App.vue           # 根组件
│   └── main.ts           # 入口文件
├── .env.development
├── .env.production
├── index.html
├── package.json
├── tsconfig.json
├── vite.config.ts
└── README.md
```

#### 2.1.2 接口对接

## 四、关键技术实现

### 4.1 Axios 拦截器
- 请求拦截器：自动添加 JWT Token
- 响应拦截器：统一错误处理，Token 过期自动刷新

### 4.2 路由守卫
- 全局前置守卫：检查登录状态，未登录用户跳转到登录页
- 路由元信息：配置哪些页面需要认证

### 4.3 Pinia 状态管理
- **userStore**: 用户信息、认证状态
- **postStore**: 帖子列表、当前帖子详情

### 4.4 日期格式化工具
- 将时间戳转换为友好显示（如：刚刚、5分钟前、1小时前等）

## 七、注意事项

1. **JWT Token 管理**：需要妥善处理 Token 的存储、刷新和过期逻辑
2. **分页处理**：帖子列表、评论列表等需要实现分页加载
3. **图片上传**：头像、帖子封面等需要实现图片上传功能
4. **错误处理**：统一的错误提示和处理机制
5. **权限控制**：某些操作（如删除自己的帖子）需要在前端进行权限校验
